# 前言

​ 此漏洞利用链为 thinkphp 5.0.x的任意文件写入的利用链,继上篇是整条文件写入利用链路上的一个任意文件删除,这篇尝试自己从入口开始挖掘分析一次任意文件写入利用链(网上的文章看着太绕了还是要自己动手分析一遍),用于理解反序列化利用链挖掘…gogogo~

# 利用链分析

block方法最终调用$this->handle的write方法,而handle我们选取 Memcached.php

接着执行Memcached对象的handler的set方法, 我们控制Memcached->handler 选取为thinkphp/library/think/cache/dirver/File.php这个类,调用的是set方法.

# 反序列化入口

入口文件为

在执行removeFiles() 方法时会调用到 file_exits ,这会导致$filename 对象的__toString方法会被调用,

跟进发现去到toArray()方法,我们的目的是执行Output类的__call类,这边选择

这个地方做跳板,那么$value,要可控,$attr要可控

这边先看$value,

$modelRelation值可以利用getError作为跳板获取.

跟进getRelationData,发现传入的$modelRelation需要Relation类型,

全局搜索下，可以利用HasOne类 **,通过x修改parent成员变量的值控制value**

接着看$attr,attr由$bindAttr的值决定, $bindAttr = $modelRelation->getBindAttr(),这里$modelRelation前面就固定死了需要一个Relation的类,而这里又调用他的$bindAttr,HasOne类去到他的父类,OneToOne类,

可以简单赋值,所以这里$attr也可控.(通过控制HasOne类的$bindAttr).
